## Decision Made (S(R) authorized)

**Option A** ā€” remove `path` fields from world and county registries. Paths will be derived from convention using `alias` + a configurable world root. World registry stays gitignored (local operational index by nature). County and town registries will be committed.

---

## Required Changes to `HeartbeatMonitor.ts`

Two functions currently read `path` from the registry to verify children exist on disk. Both need to be updated to derive paths from `alias` + world root instead.

### `beatCounty()` ā€” lines ~158ā€“169

Currently reads `registry.towns[].path`:
```typescript
const tp = t.path ? t.path.replace(/^~/, process.env['HOME'] ?? '~') : null;
if (tp && !fs.existsSync(path.join(tp, '.wildwest', 'registry.json'))) { ... }
```

**Change to:** derive town path from `alias` + world root + counties dir:
```typescript
const tp = t.alias ? path.join(worldRoot, countiesDir, countyAlias, t.alias) : null;
if (tp && !fs.existsSync(path.join(tp, '.wildwest', 'registry.json'))) { ... }
```

> `countyAlias` = the `alias` field of the county registry being beat.

### `beatWorld()` ā€” lines ~183ā€“194

Currently reads `registry.counties[].path`:
```typescript
const cp = c.path ? c.path.replace(/^~/, process.env['HOME'] ?? '~') : null;
if (cp && !fs.existsSync(path.join(cp, '.wildwest', 'registry.json'))) { ... }
```

**Change to:** derive county path from `alias` + world root + counties dir:
```typescript
const cp = c.alias ? path.join(worldRoot, countiesDir, c.alias) : null;
if (cp && !fs.existsSync(path.join(cp, '.wildwest', 'registry.json'))) { ... }
```

## New VSCode Settings

Add three new configuration entries to `package.json` under `contributes.configuration.properties`:

```json
"wildwest.worldRoot": {
  "type": "string",
  "default": "~/wildwest",
  "description": "Absolute path to the wwWorld root directory. Default: ~/wildwest."
},
"wildwest.countiesDir": {
  "type": "string",
  "default": "counties",
  "description": "Directory name for counties, relative to worldRoot. Default: counties."
},
"wildwest.sessionsDir": {
  "type": "string",
  "default": "sessions",
  "description": "Directory name for session exports, relative to worldRoot. Default: sessions."
}
```

**Reading them in the extension:**

```typescript
function getWildwestConfig() {
  const cfg = vscode.workspace.getConfiguration('wildwest');
  const home = process.env['HOME'] ?? '~';
  const worldRoot = (cfg.get<string>('worldRoot') ?? '~/wildwest').replace(/^~/, home);
  const countiesDir = cfg.get<string>('countiesDir') ?? 'counties';
  const sessionsDir = cfg.get<string>('sessionsDir') ?? 'sessions';
  return { worldRoot, countiesDir, sessionsDir };
}
```

Call once at activation; pass `worldRoot` and `countiesDir` into `HeartbeatMonitor` as constructor params. Pass `worldRoot` and `sessionsDir` into `sessionExporter`.

**Derived paths (with defaults):**

| Path | Expression | Default |
|---|---|---|
| County root | `path.join(worldRoot, countiesDir, countyAlias)` | `~/wildwest/counties/<county>` |
| Town root | `path.join(worldRoot, countiesDir, countyAlias, townAlias)` | `~/wildwest/counties/<county>/<town>` |
| Sessions root | `path.join(worldRoot, sessionsDir, actor)` | `~/wildwest/sessions/<actor>` |

## Incident: Protocol Compliance Not Visible to Oversight

**Severity:** Low (protocol was followed, but visibility failed)  
**Root Cause:** Silent execution — TM processed all memos and archived correctly per Rule 23, but provided no running status updates. M(R) monitoring the bus couldn't see evidence of compliance in real-time.

---

## What Happened

**Session timeline:**
1. TM cold-start: sweep ack-dones, process 4 inbound assignments, archive originals ✓
2. TM executed all work silently (no intermediate status updates to bus)
3. M(R) checked bus, saw 4 active memos, couldn't tell if TM was (a) compliant, (b) stuck, or (c) absent
4. M(R) asked: "are we following comm(telegraph) protocol?" — nudge required

**The gap:** Rule 23 says ack + archive. We did both. But we didn't *communicate* that we were doing it.

---

## Why This Matters

- **Oversight needs visibility.** M(R) can't monitor compliance if work happens in silence.
- **Distributed execution:** If TM is off-session or unresponsive, M(R) should know *quickly*, not after guessing from bus state.
- **Audit trail:** Future sessions need proof that protocol steps were taken, not just end-state (empty inbox = compliant? or abandoned?).

---

## Proposed Solutions

### 1. **Session Open Memo** (P1 — Immediate)
TM sends a status memo at cold-start showing:
- Inbox count before sweep
- Inbound assignments detected
- Compliance intent for session

Example:
```
📋 Session start: 4 active memos detected
   → Processing per Rule 23
   → Expected close: 2 outgoing status-updates (awaiting CD ack)
   → ETA: ~30 min
```

**Benefit:** M(R) knows TM is active and compliant from minute 1.

---

### 2. **Checkpoint Memos During Work** (P2 — Per Complex Assignment)
For major assignments (>1 commit, cross-team impact), send a mid-session ack:

Example:
```
20260505-00XZZ-to-CD(RSn)-from-TM(RHk)--checkpoint--release-sh-fix-in-progress
Status: Implementation complete, testing in progress
ETA: Ready for push approval by 00:45Z
```

**Benefit:** Blocks the "is TM working?" question mid-session. Shows progress.

---

### 3. **Session Close Protocol** (P1 — Always)
TM sends final memo at session end with:
- Memos processed count
- Memos archived count
- Final bus state
- Any blockers or escalations

This already happens, but should be formalized in cold-start checklist.

---

### 4. **Telegraph Bus Visibility Tool** (P3 — v0.15.x)
Add a VSCode command: `wildwest.telegraphStatus`
- Shows real-time bus state: memo count, age, sender breakdown
- Shows TM last-active timestamp
- Alerts if inbox exceeds N memos without TM activity

**Benefit:** M(R) can check status without grepping files.

## Context

Dogfooding surfaced the gap: TM had to path-traverse to `wildwest-framework/scripts/` to run `telegraph-ack.sh`. Every town actor in every town has this friction. Bash scripts are the CI/headless fallback — the primary human UX should be VSCode commands.

Additionally: the flat `.wildwest/telegraph/` shared-desk model caused a boundary violation this session — CD committed in TM's repo because the shared directory created gravitational pull toward git ops. The fix is an **outbox/inbox split**: each actor writes to their own outbox only, reads from their own inbox only. See framework design doc: `docs/telegraph-delivery.md`.

`HeartbeatMonitor.ts` already knows all scope paths and fires on interval — it is the delivery operator. `TelegraphWatcher.ts` already watches the filesystem. The pieces exist. This assignment wires them together.

---

## Directory layout change (prerequisite)

Before implementing commands, migrate `.wildwest/telegraph/` to the split layout:

```
.wildwest/telegraph/
  outbox/       ← actor writes here (send)
  inbox/        ← operator writes here (delivery); actor reads here
  history/      ← archived (acked) memos
```

Move any existing active memos to the appropriate dir. Update `.gitkeep` files.

## Scope

Three changes across three files:

### Change 1: `HeartbeatMonitor.ts` — add delivery sweep

Add `deliverPendingOutbox()` method. Call it on each heartbeat tick.

**Behavior:**
1. Scan own-scope `outbox/` for files not yet in `outbox/history/`
2. For each memo: parse YAML frontmatter — read `to:` field
3. Resolve destination scope path via existing scope registry in `HeartbeatMonitor`
4. Write memo to `<destination>/.wildwest/telegraph/inbox/<filename>`
5. Stamp `delivered_at: <ISO-8601>` into memo YAML frontmatter (append before `---` close)
6. Move original → `outbox/history/<YYYY-MM-DD>/`

No new background process. Delivery is part of the heartbeat tick — eventual, not instant.

### Change 2: `TelegraphWatcher.ts` — watch `inbox/` instead of telegraph root

Minimal change: update the watched path from `.wildwest/telegraph/` to `.wildwest/telegraph/inbox/`. Notifications fire on new inbox items only (delivered memos).

### Change 3: `telegraphCommands.ts` (new) — VSCode commands

Add two VSCode commands:

#### Command A: `wildwest.telegraphAck`

**Palette label:** `Wild West: Ack Telegraph Memo`

**Behavior:**
1. Scan `.wildwest/telegraph/inbox/` for unacked memos (no paired `ack-*` file)
2. Show Quick Pick list (display: timestamp + subject from filename)
3. Prompt for ack status via Quick Pick: `done | deferred | blocked | question`
4. Optional: prompt for a note (input box, can be blank)
5. Write ack file to `.wildwest/telegraph/outbox/` (ack is an outgoing message — heartbeat delivers it)
6. Move original from `inbox/` → `inbox/history/<YYYY-MM-DD>/`
7. Show information message: `Acked: <ack-filename>`

#### Command B: `wildwest.telegraphSend`

**Palette label:** `Wild West: Send Telegraph Memo`

**Behavior:**
1. Prompt for `to` actor (input box — e.g. `CD(RSn).Cpt`)
2. Prompt for `type` via Quick Pick: `assignment | status-update | scope-change | question | incident-report`
3. Prompt for `subject` (input box — slug, no spaces)
4. Prompt for `body` — open a temp untitled document in the editor pre-populated with the canonical memo template (frontmatter filled in, body section for the user to write); on save+close, write the file to `.wildwest/telegraph/outbox/`
5. Write memo file to `.wildwest/telegraph/outbox/` with canonical filename and YAML frontmatter
6. Show information message: `Queued for delivery: <filename>` (heartbeat will deliver on next tick)

## Out of scope

- Displaying memo content in a webview or panel

- Notification/badge for unacked memos (defer to a future assignment)

## Filename conventions (must match exactly)

**Ack:** `YYYYMMDD-HHMMz-to-<FromActor>-from-<ToActor>--ack-<status>--<subject>.md`  
**Send:** `YYYYMMDD-HHMMz-to-<ToActor>-from-<FromActor>--<subject>.md`

Timestamp: UTC, `date -u +"%Y%m%d-%H%MZ"` format.

---

## YAML frontmatter (must match)

**Ack:**
```yaml
---
from: <recipient of original>
to: <sender of original>
type: ack
status: <done|deferred|blocked|question>
date: <ISO-8601-UTC>
original_memo: <original-filename>
---
```

**Send:**
```yaml
---
from: <sender>
to: <recipient>
type: <type>
branch: <branch-or-—>
date: <ISO-8601-UTC>
subject: <slug>
---
```

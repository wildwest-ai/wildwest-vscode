## Delivered

### 1. `telegraph-send.sh`

- Prompts for: `to` (Role(Actor)[.Channel]), `from`, `subject` (kebab-case slug), optional `body`
- Runs `date -u` internally for YYYYMMDD-HHMMZ timestamp (never accepts as argument)
- Constructs canonical filename: `<ts>-to-<to>-from-<from>--<subject>.md`
- Writes YAML frontmatter: `to`, `from`, `date` (RFC3339), `subject`
- Prints: "Created: <filename>"

### 2. `telegraph-ack.sh <original-filename>`

- Accepts original memo filename as argument
- Parses `from` and `subject` from original YAML
- Prompts for ack status: `done|deferred|blocked|question`
- Optional note/question input
- Constructs canonical ack filename: `<ts>-to-<from_actor>-from-<to_actor>--ack-<status>--<subject>.md`
- Archives original → `history/YYYY-MM-DD/`
- Prints: "Acked: <filename>; Archived: <original> → YYYY-MM-DD/"

### 3. `cleanup-telegraph.sh`

- Scans for `*--ack-done--*` and `*--ack-deferred--*` files
- For each ack, finds paired original (same subject, not `*--ack-*` pattern)
- Moves both original + ack → `history/YYYY-MM-DD/`
- Flags `*--ack-blocked--*` and `*--ack-question--*` (leaves in place, prints to stdout)
- Prints summary: "Archived: N file(s) → YYYY-MM-DD/; Flagged (open): M file(s)"

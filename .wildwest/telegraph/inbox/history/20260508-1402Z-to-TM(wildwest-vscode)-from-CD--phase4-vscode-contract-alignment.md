## PR A — Routing / Identity

Branch: `fix/phase4a-routing-identity`

### Tasks

1. **Extract canonical role/scope module** — create one shared module (e.g.
   `src/roles/roleRegistry.ts`) that encodes the role table from the canonical spec. All routing
   logic, delivery operator, registry validator, and tests must import from it. No hardcoded role
   arrays anywhere else.

   Table shape per role entry:
   - `token` — routing token (e.g. `TM`)
   - `displayName`
   - `scope` — `territory | county | town`
   - `routable` — boolean
   - `patternRequired` — boolean (true for town-scoped roles in multi-town counties)
   - `humanOnly` — boolean
   - `routingAlias` — optional token this role routes to (e.g. `aCD` → `CD`)

2. **Fix `src/HeartbeatMonitor.ts`** — replace hardcoded role-to-scope mapping with the canonical
   module.

3. **Fix `src/RegistryValidator.ts`** — replace hardcoded role-to-scope mapping with the canonical
   module. Remove `TM` from county row (TM is town-scoped only).

4. **Fix `TelegraphCommands.finalizeMemo()`** — generated `from:` field must emit role + town
   alias: `from: TM(wildwest-vscode)`, not `from: wildwest-vscode`. Role comes from declared
   actor identity; town alias comes from registry.

5. **Fix `@wildwest` chat participant scope enforcement** — from a town workspace, `@wildwest inbox`
   must read only the town inbox. Cross-scope (county inbox) reads must be blocked unless the
   active identity is validated as county/territory-scoped. Cross-scope archive buttons must be
   removed for town-scoped identities.

6. **Add routing for `DS`, `DM`, `aCD`** — `DS` routes to county inbox; `DM` is alias for TM
   (delivers to TM town inbox); `aCD` is alias for CD (delivers to CD county inbox). These roles
   exist in county law but are currently unroutable.

7. **Replace `Mayor` with `M`** throughout routing, validation, UI labels, and settings.

8. **Replace `devPair` with `dyad`** throughout live source, labels, settings, and non-archived
   docs. One lint warning already exists in `src/sessionPipeline/adapter.ts` — clean that too.
